# 엔터티(Entity)

## 정의

엔터티란 **업무에서 관리되어야 하는 데이터의 집합**으로, 저장되고 관리되어야 하는 어떤 것(Thing)을 말한다.

> 예: 사원, 부서, 주문, 고객, 상품 등

## 엔터티의 특징

| 특징 | 설명 |
|------|------|
| 업무에서 필요성 | 업무 프로세스에서 실제로 관리해야 할 것 |
| 유일한 식별자 | 각 인스턴스를 구별할 수 있는 식별자 보유 |
| 2개 이상의 인스턴스 | 영속적으로 2개 이상의 데이터가 존재 |
| 속성 보유 | 반드시 속성을 가져야 함 |
| 관계 존재 | 다른 엔터티와 최소 1개 이상의 관계 |

## 엔터티의 분류

### 유·무형에 따른 분류

| 분류 | 설명 | 예시 |
|------|------|------|
| **유형 엔터티** | 물리적 형태가 있는 것 | 사원, 물품, 강사 |
| **개념 엔터티** | 물리적 형태 없이 개념적으로 존재 | 부서, 학과, 보험상품 |
| **사건 엔터티** | 업무 수행 시 발생하는 것 | 주문, 청구, 미납 |

### 발생 시점에 따른 분류

| 분류 | 설명 | 예시 |
|------|------|------|
| **기본(키) 엔터티** | 업무에 원래부터 존재하는 엔터티 | 사원, 고객, 상품 |
| **중심 엔터티** | 기본 엔터티로부터 발생하는 핵심 | 계약, 주문, 매출 |
| **행위 엔터티** | 2개 이상의 부모 엔터티로부터 발생 | 주문목록, 사원변경이력 |

## 엔터티 명명 규칙

1. 업무에서 사용하는 용어 사용
2. **가능하면 약어 사용 금지**
3. 단수 명사 사용
4. 유일한 이름 부여
5. 생성 의미대로 이름 부여

## 예제: ERD 표기

```
┌────────────────┐          ┌────────────────┐
│     사원        │─────────│     부서        │
├────────────────┤          ├────────────────┤
│ 사번 (PK)      │          │ 부서코드 (PK)  │
│ 사원명         │          │ 부서명         │
│ 부서코드 (FK)  │          │ 위치           │
│ 입사일         │          └────────────────┘
└────────────────┘
```

## 시험 포인트

- 엔터티 특징 5가지 암기 (업무필요성, 유일식별자, 2개이상인스턴스, 속성보유, 관계존재)
- 유형/개념/사건 엔터티 구분 예제
- 기본/중심/행위 엔터티 발생 순서 이해
