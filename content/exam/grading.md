# SQLP 채점 기준

## 점수 배분

| 과목 | 객관식 | 단답형 | 서술형 | 소계 |
|------|--------|--------|--------|------|
| 1과목: 데이터 모델링의 이해 | 10문항×2점 | - | - | 20점 |
| 2과목: SQL 기본 및 활용 | 20문항×2점 | - | - | 40점 |
| 3과목: SQL 고급활용 및 튜닝 | 10문항×2점 | 5문항×2점 | 5문항×4점 | 40점 |
| **합계** | | | | **100점** |

> ⚠️ 위 배분은 일반적인 예시이며, 실제 시험 공고를 반드시 확인하세요.

## 합격 기준

```
✅ 합격 조건:
  1. 전 과목 평균 75점 이상
  2. 각 과목 40점 이상 (과락 없어야 함)
```

## 객관식 채점

- 정답 1개 선택
- 오답 시 0점 (감점 없음)

## 단답형 채점

- 정확한 용어/SQL 구문 작성 필요
- 부분 점수 없음 (맞거나 틀리거나)
- 대소문자 구분 없음 (일반적으로)

## 서술형 채점

- 부분 점수 있음
- 핵심 키워드 포함 여부 중요
- 논리적 흐름과 정확성 평가

## 3과목 서술형 예시 유형

1. **SQL 실행 계획 분석**: 주어진 실행 계획을 보고 문제점 파악 및 개선 방안 제시
2. **인덱스 설계**: 특정 쿼리에 최적화된 인덱스 설계
3. **SQL 튜닝**: 비효율적인 SQL을 튜닝하여 개선된 SQL 작성
4. **데이터베이스 구조 분석**: Lock, 트랜잭션 관련 문제 분석

## 공부 전략

| 과목 | 비중 | 전략 |
|------|------|------|
| 1과목 | 20% | 개념 정확히 이해, 용어 암기 |
| 2과목 | 40% | SQL 실습 병행, 함수/조인 완벽 이해 |
| 3과목 | 40% | 실행계획 읽기 연습, 튜닝 원리 이해 |

> 💡 3과목 서술형에서 점수를 많이 따야 합격이 수월합니다. 실제 쿼리 작성 연습을 많이 하세요.
